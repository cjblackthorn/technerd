# TechNerd Site Restructure Phased Plan

## Direction

TechNerd Consulting is moving from a broad practical technology site into an enterprise-informed cybersecurity governance practice. The public hierarchy should make enterprise security, identity governance, audit readiness, and risk work primary, while preserving small-business and nonprofit services as a right-sized secondary path.

Core message:

> Enterprise security experience. Practical solutions for organizations of every size.

## Phase 1: Foundation and Content Models

Goals:

- Add scalable Jekyll collections for services, packages, projects, insights, resources, and frameworks.
- Move primary navigation into structured data.
- Add reusable layouts that can render structured front matter without bespoke page code.
- Add flexible pricing fields for packages and services without hard-coding pricing into components.
- Preserve the current visual identity while making the information architecture more mature.

Deliverables:

- Collection configuration in `_config.yml`
- Data-driven navigation in `_data/navigation.yml`
- Reusable collection layouts
- Initial MVP content entries for services, packages, projects, insights, and resources

## Phase 2: Core Public Restructure

Goals:

- Rewrite the homepage around enterprise security, governance, identity, and risk.
- Rebuild Services as a structured overview with audience paths and engagement models.
- Update About to support enterprise credibility without reading like a resume.
- Add public index pages for Insights and Projects.
- Refocus Resources around practical downloadable tools and checklists.

Deliverables:

- Updated homepage
- Updated About page
- Updated Services overview
- Insights index
- Projects index
- Resources index
- Updated Contact/CTA flow

## Phase 3: Service Catalog and Offers

Goals:

- Expand service pages into clear, discoverable engagement categories.
- Add package pages with scope, deliverables, timeline, client responsibilities, and pricing display fields.
- Add a la carte service architecture after package structure is stable.

MVP service groups:

- Enterprise Security & Governance
- Strategic Advisory
- Small Business & Nonprofit Security
- Service Packages
- A La Carte Services

MVP packages:

- Security Foundations Assessment
- Identity & Access Health Check
- Audit Readiness Assessment

## Phase 4: Proof Layer

Goals:

- Publish anonymized or composite enterprise case studies that demonstrate substantial bodies of work.
- Tie projects back to services, insights, and resources.

MVP projects:

- Enterprise External Active Directory Governance & Cleanup
- Azure Storage & Key Vault Governance
- IAM Audit Support

## Phase 5: Cornerstone Content

Goals:

- Publish a small set of high-authority insights before expanding volume.
- Use each article to support services, packages, projects, and resources.

Initial insights:

- Identity Governance Is a GRC Function, Not Just an IT Function
- Evidence Is Part of the Control
- Seven Reasons Access Reviews Fail Audits

## Phase 6: Resource Library

Goals:

- Add no-login practical resources that demonstrate expertise and create value.
- Link resources to services, packages, insights, and projects.

Initial resources:

- Access Review Checklist
- Audit Evidence Checklist
- Identity Governance Assessment
- Application Ownership Matrix

## Phase 7: Frameworks and Visual Models

Goals:

- Build original TechNerd intellectual property over time.
- Add simple diagrams and models that can support articles, proposals, and LinkedIn content.

Initial framework targets:

- Identity Governance Maturity Model
- Audit Readiness Framework
- Evidence Quality Framework
- Application Ownership Framework

## Phase 8: SEO, Accessibility, and Structured Data

Goals:

- Expand metadata for content types.
- Add structured data where appropriate.
- Validate WCAG 2.2 AA practices where practical.
- Improve internal linking and breadcrumbs.

## Phase 9: Productization and Distribution

Goals:

- Prepare for future paid templates, workshops, retainers, and digital toolkits without building commerce prematurely.
- Add cross-publishing support for Blackthorn Field Notes and LinkedIn adaptations.

Deferred:

- Shop
- Client portal
- Paid downloads
- Online assessments
- Membership resources
