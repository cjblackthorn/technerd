# Request Consultation

TechNerd Consulting uses a structured consultation and discovery workflow designed to understand project goals, technical requirements, accessibility considerations, and operational needs before work begins.

## Helpful Information

When requesting a consultation, please include:

- **Your Name** — The name you would like TechNerd Consulting to use when responding.
- **Organization or Business Name** — If the request is for a business, nonprofit, team, or organization.
- **Project Type** — For example: IT support, cybersecurity guidance, accessibility consulting, systems planning, workflow improvement, or troubleshooting.
- **Accessibility Needs or Considerations** — Any access needs, device limitations, mobility needs, sensory needs, communication preferences, or workflow barriers that should be considered.
- **Project Goals** — What you want the technology, workflow, or system to accomplish.
- **Desired Timeline** — Any deadlines, urgency, scheduling limits, or preferred timing.
- **Budget Range** — If appropriate, a general range helps determine whether the request can be scoped realistically.

## Review Process

Requests are reviewed to assess:

- **Project Fit** — Whether the request falls within TechNerd Consulting’s current service areas.
- **Scope** — The size, complexity, and likely boundaries of the work.
- **Technical Requirements** — Systems, devices, platforms, access, accounts, tools, or documentation needed.
- **Accessibility Considerations** — Barriers, accommodations, usability needs, or adaptive workflow concerns.
- **Timeline Expectations** — Whether the desired schedule is realistic for the request.
